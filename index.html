Lesson 1: The Basics of the Web and HTML

How the Web Works

To many the Web is a mystery that can only be unlocked by a nerd. But how the Web works can be quite facinating for most people. Basically it's a huge network of computers talking to each other. When you request a web page your computer sends a request to a server that sends the requested document.

The problem is that computers don't speak English. The main language for the Web is HTML (Hypertext Markup Language). HTTP is the "grammer" of the Web. Urls are the addresses for the pages and files you are looking for. The Web was created in the early 1990's. It currently has about 30 billion pages.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML is made up of:

  1. Text Content - What you see
  2. Markup - What it looks like
  3. References to other documents e.g. images and videos
  4. Links to other pages

This video gives a good overview.

Tags and Elements

HTML documents are made of HTML elements. An element is made up of an opening tag, contents (the information you want to convey), and a closing tag. The opening tag can also contain attributes that assist in coding and style.

This video explains the differences.

Why Computers are like Spock

Spock is very logical and also very literal. Agonozingly so. Computers are just like that. Computers don't like mistakes. Everything has to be exactly as the computer wants it or you get ugly error messages back. Computers like to remind us that we're human - just like Spock! Therefore, Spock must be a COMPUTER.

Inline or Block?

Some HTML elements just can't get out of the box.

  1. p
  2. div
  3. h1 and h2
  4. and many other tags form an invisible box (or block) around the contents

On the other hand, other tags are strickly inline. Like for instance the span tag.

Structure from Chaos: "Treeing" HTML

When it comes to HTML, being in the box is a good thing. Some HTML elements can contain many other HTML elements which in turn can contain many other elements, and so on, and so on...

This nesting of boxes gives structure to the code and lends itself to the "tree" analogy. Personally, I think you need a few if/then/else statements before things start "branching" out.

So What's in the box?

If you are doing a good job of writing code then you indent as you go deeper into the "nested boxes". The greater the indentation, the deeper in the "nested boxes" you are. If you don't indent when you write code then good luck figuring out what you have or debugging it! You will end up getting Lost in Space....

Lesson 3: Turning plain HTML into a CSS Rock Star

Avoiding Repetition

No one likes to write or read the same code over and over. It's boring and it takes up time and space. More than that it means more errors and more debugging. Even more, what happens when your boss says he wants a differnt font used - and it's in fifty places in your code!! HTML has an answer for that -

DEFINE CLASSES. This keeps you from repeating code over and over. If you need to make a change then it happens once in the class definition. I think that's what Object Oriented Programming is all about.....

Adding the CSS touch to HTML

CSS (Cascading Style Sheets) is what makes HTMl look good. This is a seperate language that addresses all the style issues that HTML has. The HTML must link to the corresponding CSS file. This gives the programmer the ability to change the way a page looks without changing any HTML. It also greatly assists in creating a common "theme" among related webpages.
